### Qu'est-ce que ShadCN UI ?

<img src="/img/blog/shadcn/dashboard.jpg" alt="ShadCN UI Logo" />

[![ShadCN UI Logo](/img/blog/shadcn/dashboard.jpg)](/img/blog/shadcn/dashboard.jpg)

ShadCN UI est une bibliothèque de composants UI (interface utilisateur) qui se distingue par sa simplicité et son efficacité. Conçue pour être parfaitement compatible avec Tailwind CSS, cette bibliothèque offre une collection de composants prêts à l'emploi, élégants et réactifs. L'objectif principal de ShadCN UI est de permettre aux développeurs de créer des interfaces utilisateur modernes sans avoir à réinventer la roue à chaque projet.

### Pourquoi Utiliser ShadCN UI ?

1. **Intégration Parfaite avec Tailwind CSS** : ShadCN UI est conçu pour tirer pleinement parti des capacités de Tailwind CSS. Cela signifie que vous pouvez facilement personnaliser les composants en utilisant les classes utilitaires de Tailwind, ce qui vous permet de maintenir un style cohérent tout en conservant la flexibilité nécessaire pour des ajustements rapides.

2. **Productivité Accrue** : En utilisant des composants préconstruits et bien conçus, vous pouvez réduire considérablement le temps passé à créer et à styliser des éléments de base de votre application. Cela vous permet de vous concentrer sur les aspects les plus complexes et uniques de votre projet.

3. **Composants Modernes et Réactifs** : Les composants de ShadCN UI sont non seulement esthétiquement plaisants, mais ils sont également optimisés pour la réactivité. Que vous développiez pour un bureau ou un appareil mobile, vous pouvez être assuré que vos interfaces s'adapteront parfaitement à différentes tailles d'écran.

4. **Documentation Complète et Communauté Active** : ShadCN UI est accompagné d'une documentation détaillée qui facilite son adoption et son utilisation. De plus, une communauté active de développeurs contribue régulièrement à son amélioration, partage des astuces et résout des problèmes communs.

### Comment Commencer avec ShadCN UI ?

Commencer avec ShadCN UI est un processus simple. Voici les étapes de base pour intégrer cette bibliothèque dans votre projet Tailwind :

1. **Création d'un nouveau projet Vite** :


   ```bash
   npm create vite@latest
   ```

2. **Ajouter Tailwind et sa configuration**

###### Installation de tailwindcss et de ses dépendances, puis génération de vos fichiers de configuration tailwind.config.js et postcss.config.js :

```bash 
npm install -D tailwindcss postcss autoprefixer

npx tailwindcss init -p
```

3. **Modifier le fichier tsconfig.json**
###### Ajoutez le code suivant au tsconfig.jsonfichier pour résoudre les chemins :

```tsx
{
  "compilerOptions": {
    // ...
    "baseUrl": ".",
    "paths": {
      "@/*": [
        "./src/*"
      ]
    }
    // ...
  }
}
```

4. **Mettre à jour vite.config.ts**

###### Ajoutez le code suivant à vite.config.ts pour que votre application puisse résoudre les chemins sans erreur

```bash
# (so you can import "path" without error)
npm i -D @types/node
```
```tsx
import path from "path"
import react from "@vitejs/plugin-react"
import { defineConfig } from "vite"

export default defineConfig({
  plugins: [react()],
  resolve: {
    alias: {
      "@": path.resolve(__dirname, "./src"),
    },
  },
})
```

5. **Exécuter la CLI**

```bash
npx shadcn-ui@latest init
```

6. **Configurer les composants.json**

###### Quelques questions vous seront posées pour configurer **components.json**:

```bash
Would you like to use TypeScript (recommended)? no / yes
Which style would you like to use? › Default
Which color would you like to use as base color? › Slate
Where is your global CSS file? › › src/index.css
Do you want to use CSS variables for colors? › no / yes
Where is your tailwind.config.js located? › tailwind.config.js
Configure the import alias for components: › @/components
Configure the import alias for utils: › @/lib/utils
Are you using React Server Components? › no / yes (no)
```

7. **Utilisation des Composants** : 
###### Une fois la configuration terminée, vous pouvez commencer à utiliser les composants de ShadCN UI dans vos fichiers TSX. Par exemple, pour ajouter un bouton stylisé :


```bash
npx shadcn-ui@latest add button
```

   ###### La commande ci-dessus ajoutera le Buttoncomposant à votre projet. Vous pouvez ensuite l'importer comme ceci :
   ```tsx
   import { Button } from "@/components/ui/button"

export default function Home() {
  return (
    <div>
      <Button>Click me</Button>
    </div>
  )
}

   ```
