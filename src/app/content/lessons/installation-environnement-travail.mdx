# Installation de l'environnement de travail

Avant de commencer à coder, nous devons configurer notre environnement et installer les outils nécessaires pour travailler efficacement.

## Installation de Visual Studio Code

Le premier outil que nous allons installer est Visual Studio Code (VSCode), un IDE (Integrated Development Environment). C'est l'endroit où nous allons écrire notre code et construire nos projets.

---

> ❓ **Pourquoi Visual Studio Code ?**
>
> Visual Studio Code est l'IDE le plus reconnu pour le développement web et il est 100% gratuit. De plus, il propose de nombreuses extensions qui facilitent et optimisent le codage.

---

### Installation de Visual Studio Code

Pour installer VSCode :

1. Recherchez Visual Studio Code sur internet ou allez directement à l'URL suivante : [Lien d'installation de VSCODE](https://code.visualstudio.com/)
2. Cliquez sur “Download” en haut à droite, puis choisissez la version correspondant à votre système d'exploitation (Mac, Linux ou Windows). Le téléchargement devrait commencer automatiquement, et vous serez redirigé vers la documentation.
3. Rendez-vous dans votre dossier de téléchargements et complétez l'installation en suivant les instructions.

---

> 💡 **Conseil pratique :** La documentation officielle de VSCode est très utile. N'hésitez pas à la parcourir pour approfondir vos connaissances.

---

### Extensions pour Visual Studio Code

Une fois VSCode installé, nous allons installer quelques extensions pour faciliter le développement de nos projets :

1. **Live Server** : Cette extension lance un serveur local qui se rafraîchit à chaque sauvegarde du code.
2. **Auto Rename Tag** : Cette extension renomme automatiquement la balise HTML fermante correspondante lorsque vous modifiez la balise ouvrante.

Pour installer ces extensions :

1. Ouvrez VSCode.
2. Cliquez sur l'icône des extensions dans la barre latérale gauche.
3. Recherchez les extensions par nom (Live Server et Auto Rename Tag) et cliquez sur "Installer".

---

> 💡 **Note :** Les extensions peuvent considérablement améliorer votre productivité. Explorez le marketplace des extensions pour découvrir d'autres outils utiles.

---

### Autres extensions recommandées

Voici quelques autres extensions que vous pourriez trouver utiles :

1. **Prettier - Code Formatter** : Formate automatiquement votre code pour qu'il soit propre et cohérent.
2. **ESLint** : Identifie et corrige les problèmes dans votre code JavaScript en suivant les règles définies.
3. **Debugger for Chrome** : Permet de déboguer votre code JavaScript directement dans Chrome.
4. **Path Intellisense** : Autocomplète automatiquement les chemins de fichiers dans votre projet.
5. **Bracket Pair Colorizer** : Colorie les paires de parenthèses, crochets et accolades pour une meilleure lisibilité du code.

Pour installer ces extensions, suivez le même processus décrit précédemment en recherchant chaque extension par nom dans le marketplace des extensions.

## Installation de Git

---

> ❓ **Qu'est-ce que Git ?**
>

## Premiers pas avec Git

### Initialiser un dépôt Git

Pour commencer à utiliser Git avec votre projet, vous devez initialiser un dépôt Git dans le répertoire de votre projet.

1. **Ouvrez VSCode** et accédez au terminal intégré (`Ctrl + ` ou `Cmd + `).
2. **Naviguez** vers le répertoire de votre projet avec la commande `cd` :
   
   ```bash
   cd chemin/vers/votre/projet
   ```

3. **Initialisez** un dépôt Git avec la commande suivante :
   
   ```bash
   git init
   ```

Cette commande crée un nouveau sous-répertoire `.git` dans votre projet, qui contient tous les fichiers nécessaires au suivi des modifications.

### Faire un commit initial

Après avoir initialisé le dépôt, il est temps de faire notre premier commit.

1. **Ajoutez** les fichiers de votre projet à l'index (staging area) avec la commande :

   ```bash
   git add .
   ```

   Le point `.` indique que tous les fichiers dans le répertoire actuel doivent être ajoutés à l'index.

2. **Créez** un commit avec la commande :

   ```bash
   git commit -m "Initial commit"
   ```

   L'option `-m` permet de spécifier un message de commit décrivant les modifications. Ici, "Initial commit" est le message que nous utilisons.

---

> 💡 **Conseil pratique :** Les messages de commit doivent être clairs et descriptifs pour que vous et vos collaborateurs puissiez comprendre facilement les modifications apportées.
